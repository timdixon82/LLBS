# LLBS: Living Well Together Strategy and Photo Brander

This repository holds two web deliverables for the Lincoln and Lindsey Blind Society (LLBS).

## Live site

The repository is published at [timdixon82.github.io/LLBS](https://timdixon82.github.io/LLBS). The two pages are:

- [Living Well Together Strategy](https://timdixon82.github.io/LLBS/index.html): the public strategy page.
- [LLBS Photo Brander](https://timdixon82.github.io/LLBS/brand.html): the social media image tool for staff and volunteers.

## What the project is

### Strategy page (index.html)

`index.html` presents the LLBS Living Well Together Strategy 2025 to 2030 as a public web page. It is a read-only information page with in-page navigation, key statistics, a Strengths, Weaknesses, Opportunities, and Threats (SWOT) analysis, vision and mission statements, organisational values, four strategic themes with a tabbed detail view, a timeline, income targets, alignment with the Lincolnshire Joint Health and Wellbeing Strategy (JHWS), and a call to action.

The page serves members, funders, commissioners, volunteers, partner organisations, and the public. It includes people who are severely sight-impaired and who rely on screen readers such as JAWS and VoiceOver.

### Photo Brander (brand.html)

`brand.html` is a browser-based tool that lets LLBS staff and volunteers create branded social media images. A user uploads a photo, positions it in the frame, adds optional overlay text, and downloads or shares the finished image. All image processing happens inside the browser using the Canvas Application Programming Interface (API). No photo or user data is sent to any server.

## How to run it locally

The project has no build step. Both pages are served directly as static files.

Serve the repository root with any static file server. The simplest option on most systems is Python 3:

```
python3 -m http.server 8080
```

Then open `http://localhost:8080` in a browser.

Alternatively, use the Node.js `serve` package:

```
npx serve .
```

Or use the VS Code Live Server extension, or any other static file server.

## Setting up the development tools

The linting tools (ESLint, Stylelint, and html-validate) are listed as development dependencies in `package.json`. To install them:

```
npm install
```

After installing, copy the DM Sans font files into `assets/fonts/`:

```
npm run copy-fonts
```

Then run the linters:

```
npm run lint
```

The `package-lock.json` file, which locks dependency versions, is not committed. It is generated by `npm install`. The LLBS logo image files in `assets/` are not distributed with this repository. See `assets/PLACEHOLDER.md` for download instructions.

## Accessibility

Both pages are built to WCAG 2.2 at AAA conformance. That is the highest of the three WCAG levels. It satisfies the accessibility laws in scope: the United Kingdom Equality Act, the Public Sector Bodies Accessibility Regulations, the European Accessibility Act, the Americans with Disabilities Act, and Section 508.

The team tests with VoiceOver on macOS and JAWS on Windows.

Documented accessibility exceptions are in `docs/exceptions/`. Each exception records the specific gap, the rationale, and the approval date.

## Security

Each page carries a Content-Security-Policy (CSP) delivered via a meta tag and a Referrer-Policy meta tag. Two security headers that require an HTTP response header (X-Frame-Options and Permissions-Policy) cannot be delivered on GitHub Pages; this gap is documented as an accepted exception in `docs/exceptions/SEC-LLBS-001-security-headers.md`.

Web fonts (DM Sans) are self-hosted under `assets/fonts/`. Both LLBS logos are self-hosted under `assets/`. No external font service or third-party image host is contacted at load time.
